タチコマMCP 思考戦車 · 6機配備 · 2ホスト

2 つのホストにまたがって運用される、AI 強化型のスパイダー戦車 6 機。各機は単一の能力を EXEC_MODE=local プロトコルで包んでいる — Hermes Gateway がリクエストをディスパッチし、タチコマは SSH を往復することなくローカルで解決する。ほとんどはディープダイブ・ノード（Kali VM）で走る；もっとも人気のある一機はオペレータの Windows ホストで走る、なぜならVault がそこに住んでいるから。

02 連携ドクトリン

タチコマは状態を共有しない。各機は純粋関数：入力 → 出力、記憶なし。記憶は彼らをオーケストレーションする電脳化部（Batou）、それからHoncho dialectic（長期パターン）、そしてVault（ナレッジベース）に住む。これは意図的 — 各ユニットを置換可能、テスト可能、独立しているが共にある状態に保つ。

Hermes Gateway がディスパッチ・テーブルを持つ。Batou が cve-lookup.query("CVE-2026-12345") を呼ぶと、Hermes はそれを Kali 上のローカル stdio ハンドルに解決する、SSH の往復はない。shared/src/ssh.mjs の EXEC_MODE=local パッチが、タチコマが Gateway と同じ場所にいるときに SSH 層をバイパスする。パッチは Tier 4-P リファクタのあいだに適用された。

「私たちはスタンドアロンだけど、一緒にいる。」 — タチコマ // Stand Alone Complex（言い換え）

03 採用 // 空きスロット

現在 1 つのスロットが空いている（V-006）。候補のタチコマを現役勤務に昇格させる条件：

  • · 単一の能力、よく定義されていること。
  • · ステートレス — 他のユニットと共有された可変状態を持たないこと。
  • · ローカル実行が望ましい（ネットワーク往復を避ける）。
  • · ~/logs/<unit>.log に構造化 JSON でログ出力すること。
  • · Hermes のシャットダウンに対して優雅に死ぬこと。

現在検討中の候補：金融/スプレッドシート読み取り、カレンダー・ブリッジ、arXiv 取り込みヘルパー。まだ昇格はゼロ — オペレータのドクトリンは 追加する前にシンプル化する である。
